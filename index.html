CSS ID Selectors

Class selectors which we did just previously, allow us to beautify HTML elements of specific classes. 'Anchor elements' allow us to style individual elements uniquely and let other elements go unharmed. This takes an element like 'p' with 'id' as the attribute. An ID attribute can only be used once because it is unique to the element it is assigned to. If there is a CSS rule for a class attribute, the id attribute will over-rule it. To create a rule for an attribute in CSS you use a '#' instead of a '.' If unsure between using 'id' or 'class' selectors, this page may be helpful.

Boundaries and Space

This is all about organising a webpage using the 'Box Model' for aesthetic reasons. Essentially it's boxes in boxes in boxes, with each box centered within the previous and works inward in order like this:

  1. Margin - First box set with the 'margin' property; white space between elements
  2. Border - 2nd box set with the 'border property; an outline
  3. Padding - 3rd box - gives extra space between the content and the border
  4. Content - Text, images, media etc

Borders

Not really sure why they didn't just include this in the 'Styling CSS' component, but okay. I am working on borders now. It's basically a picture-frame around content. A 'border' property takes 3 values:

  • Thickness - Self-explanatory. It takes rem, em and px.
  • Type - Border style which may be solid, dotted or dashed.
  • Color - Again, using, named, HEX or RGB

You can also change these individually using the properties 'border-style', 'border-color' and 'border-width'. Originally, I was using the 'margin-left' etc properties to set a border around page, but I will now use the border property.

Padding

Padding is a great way to avoid adding margins to just about everything. It's basically the space between an element's content and its border. Increasing the white space improves readability. Just like 'margin', you can also individually alter each side of padding using padding-left, right, etc, or you can input 4 values (one for each side) in the 'padding' property going from left to right, the order is top, right, bottom, left (clockwise).

I have now added padding to my paragraphs and it has caused a bit of mayhem and uneveness which hopefully I can resolve.

Position

The final lesson here is all about the 'position' property in CSS which allows us to place our html elements into precise locations we want them to be. We can use the value 'relative' to position page elements on a webpage relative to where they would normally appear. By first setting position: relative, you can then use the CSS properties top, left, bottom, and right to shift an element away from where it would have normally appeared on the page. We can even use 'position' to create 'clickable buttons'.

Click Me!

To do this, I needed to have 3 selectors:

  1. Button - To manipulate margins
  2. Button a - To create a border, padding, position and cursor properties
  3. Button a:active - To create the click bounce movement thing using the 'top' property with a px value

Other position values which use properties like top, right, left and bottom include:

  • Position:Absolute - positioned in relation to the nearest positioned parent element. If no positioned parent element, the element is positioned in relation to the html element
  • Position:Fixed - the element is fixed in position and won't move even when scrolling
