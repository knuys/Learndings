Variables and Root

Variables

Variables in CSS are denoted using '--' before the variable name. This would allow you to change multiple style properties at once. You can reference your variable using 'var(--variable-name)' as a value to a property. You may also assign a Fallback value.This will be displayed if the desired value or variable could not be found. Good for debugging or compatibility. This can look like:

  • background:var(--variable-name, fallback);
  • or simply background:red; if placed the line above the variable.

Root

When you create a variable, it is available for you to use inside the selector in which you create it. It also is available in any of that selector's descendants. This happens because CSS variables are inherited, just like ordinary properties. To make use of inheritance, CSS variables are often defined in the :root element. :root is a pseudo-class selector that matches the root element of the document, usually the html element. By creating your variables in :root, they will be available globally and can be accessed from any other selector in the style sheet. Define a variable named --penguin-belly in the :root selector and give it the value of pink. You can then see that the variable is inherited and that all the child elements which use it get pink backgrounds. When you create your variables in :root they will set the value of that variable for the whole page. You can then over-write these variables by setting them again within a specific element.

Media Queries

CSS Variables can simplify the way you use media queries. For instance, when your screen is smaller or larger than your media query break point, you can change the value of a variable, and it will apply its style wherever it is used. In the :root selector of the media query, change it so --penguin-size is redefined and given a value of 200px. Also, redefine --penguin-skin and give it a value of black. Then resize the preview to see this change in action.

Position

We can use this to override the CSS box-model and place items where we would like them. We will be looking at the following 4 items:

  • Relative - this allows placement of elements in a spot relative to the normal flow in the box model. This uses the CSS offset properties of left or right, and top or bottom to say how far away from the normal position the element will now be.
  • Absolute - this locks an element into a position relative to the parent container. This takes the element out of the normal flow of the document making surrounding items ignore it. Basically, it's out here doing its own thing and everyone else is none the wiser. The CSS offset properties are used to adjust the position.

    • Be careful! Absolute positioning can be problematic. If you don't add a position rule to your parent, your browser will look 'up the chain' to prescribe your 'absolute'. It could result in defaulting to the body tag to receive that info.

  • Fixed - a type of absolute position. This locks an element into place relative to the browser window and takes CSS offset properties. Fixed allows you to have something stay in the same position even when you scroll. However, be careful with styling - as it is removed from the normal flow of the document, it can disrupt the aesthetics of the page resulting in the need for layout adjustments/accommodations. It doesn't occupy a 'space' per se and just hangs around on a spot on the viewport - like a fly on your screen.
  • Sticky - a type of relative position similar to fixed. However, it does occupy a space in a container, and acts similar to fixed once it is scrolled past a defined position for it. You can observe this below in my triptych of my favourite animal crossing friends.

I have a background in art and am an avid painter. So colour theory is something I have a lot of experience with, but it doesn't mean it is easier for me. For a thesis project - having the appropriate colours for my presentation not only helped to explain my project, but also improved the professional look of my poster. It helped to significantly increase engagement with my project relative to my peers. All this is telling you is that colour and design really matters!

A good grasp of colour theory will really help your projects pop. So far, I've gone for very neutral colours that have stayed within the same colour family as this is a journal. However, there may be some instances where I would need to really play with colour theory to evoke different emotions,make elements eye-catching and influence the overall general feel of my project. In keeping within the same colour family, I've helped my journal look clean, unfussy and organised, but it's not going to help me win any prizes. Useful tools include the colour-wheel, checking out Pantone layouts and drawing inspiration from source material related to your project. But let's first talk about the colour wheel.

  • Colours on the opposite sides of the wheel are complementary. These are great for creating high contrast and making elements stand-out from each other. Artist Takashi Murakami uses complementary colours a lot in his art to make his works vibrant. Mixing complementary colours will create a gray.
  • Neighbouring colours are different hues from a similar source - you can think of these colours as belonging to the same family. For example the colours to the left and right of red are shades of red with more yellow (clockwise) and more magenta/blue (anti-clockwise) pigment. Using these colours together ensures harmony as you are selecting colours within the same family.
  • Adding varying levels of white to a colour produces lighter tints of the same colour.
  • Adding varying levels to a colour produces darker shades of the same colour.
  • Adding warm tones (red/yellow/orange) to a colour will create a warmer toned version of the colour.
  • Adding cool tones (magenta/blue/green) to a colour will create a cooler toned version of the colour.
Modern colour theory follows the additive RGB model which is used in digital devices, and the subtractive CMY(K) model which is predominantly used in printing. To help you with choosing colours, you can use a colour picking tool. The example below examines the basic principles of colour theory using red.

Tertiary Colours

In the RGB model, the primary colours are red, green and blue. Mixing two primary colours results in secondary colours. And mixing a primary colour with a secondary colour results in a tertiary colour. As shown below, mixing blue and green creates cyan, blue and red creates magenta, and red and green creates yellow. Seconary colours are also complementary or contrasting colours to the colour they do not contain - for example cyan complements red as blue and green create cyan, and not red.

Although contrast is great for impact, too many contrasting colours can be overwhelming. Putting text on a contrasting colour can make it annoying to read as well. Thus it is best to use just one contrasting colour and delegate it towards creating visual impact for an element you wish to highlight.

Applied Accessibility

Ensuring that your site is able to be experienced by those with low vision or who use screen-readers is important. Basically, assistive technologies can analyse a webpage for specific tags as the reader may set the reader to scan for specific areas to jump to and read areas which are then relevant for them. For example, the reader may be set to only read main headings like 'h1', thus if you don't use that in your code, your headings won't be read. This is referred to as 'semantic meaning' - making your code semantically meaningful is really useful for making your page accessible. One of the examples was that a description of an image used as a page break should not be included in 'alt'. And although HTML5 treats these tags no different to a div, they provide a lot of semantic information to assistive technologies that can optimise their user's experience of your site. Here are a few more tags that are helpful for those who use screen readers:

  • 'main' - This wraps around the main content of your page and does not include elements that repeat across the site like headers, navs and footers. This tag gives assistive tech the ability to 'jump to main' for example.
  • 'article' - Groups independent, self-contained content. Wraps around independent sections of content such as blog entries, forum posts or news articles. To determine what fits together, ask yourself if you removed all surrounding context, would that content still make sense?
  • 'section' - Groups related content. Wraps around thematically related content rather than independent content, like a chapter to a book.
  • 'div' - Groups content. Use this for content where there is no relationship between it and other groups. There is no point in sectioning it into a section or article basically.
  • 'nav' - Wraps around the main navigation links of the page. If there are recurring links acrossing pages, just stick them in a header or footer. Nav is more for page navigation, not site-wide navigation.
  • 'figure' - Wraps around a figure (i.e image, chart, diagram etc). This is paired with 'figcaption' internally.
  • 'figcaption - Goes with 'figure' and replaces the 'p' element. It is for captioning the figure.
  • 'label' - Wraps around the text for a specific form control item, usually the name or label for a choice. Previously we used 'label' for radio buttons where we wrapped the radio button input element inside a label element along with the label text in order to make the text clickable. Another way to achieve this is by using the for attribute as explained in this lesson. The value of the for attribute must be the same as the value of the id attribute of the form control.
  • 'fieldset' - Wraps around a group of radio buttons which are possible answers to one question. It also uses a 'legend' tag which describes the grouping - basically replaces the 'p' tag/what is used to express the question. 'fieldset' and 'legend' aren't necessary if the choices are self-explanatory like in gender selection.
  • 'input' - Used to provide form controls for a variety of input types. You can specify these types using the 'type' attribute - types may include 'submit', 'date' and 'text' - all fairly self-explanatory. 'text' is the default type. Don't forget to 'name' your input!
  • 'time' and 'datetime' attribute - This element + attribute pairing can wrap around text that allows us to express and produce the more specific and non-colloquial version of it to avoid confusion inline for screenreader users.
  • 'accesskey' - Lets you assign a keyboard shortcut key to an element which when hit will allow you to activate or focus on the element. This is case sensitive.
  • 'tabindex' - Lets someone tab to an element. The value (positive, negative and 0) determines the behaviour. You can use the CSS pseudo-class ':focus' with it. You can use 'tabindex' to cycle through the order of tabbing using the value, the lowest number from 1 being first. 'tabindex' will be explored further later on.

Aside from tagging your page, there are other considerations to be made. Such as how are you going to have screenreaders read a chart or diagram? Well you are better off using a table for them, but what if you really want that graph and not both? Well you can hide the table from non-screenreader using users by playing with CSS by setting 'position:absolute', width/height to 1px and left to -10000px.

The following do not work, despite what people online may say:

  • display: none; or visibility: hidden; hides content for everyone, including screen reader users
  • Zero values for pixel sizes, such as width: 0px; height: 0px; removes that element from the flow of your document, meaning screen readers will ignore it

Other good ideas from the Web Content Accessibility Guidelines include:

  • High contrast text - Recommended to be at least a 4.5 to 1 contrast ratio for normal text. This ratio is calculated by comparing the luminance values of two colours which can range from 1:1 = same colour/no contrast, to 21:1 which = white vs black/the most contrast. You can use online tools to check for contrast ratios.
  • Avoid colourblindness - You can counteract the muddy/blurring greyscale from colourblindness by ensuring you have strong enough contrast. A 4.5:1 is advised - this can be done by shading the darker colour and tinting the lighter colour.
  • Choose the right colours - Maybe don't choose colours too close to each other on the colour wheel in the same space or as a foreground vs background colour. Use a colour calculator or colour simulator if you are having trouble.

Media Queries

Media queries basically adjusts content to fit into the viewport. The viewport is essentially the dimensions of visible page space the user can see, which is dependent upon the device they are using. Media queries use a media-type, which if it matches with the device the page is displayed on, the relevant styles will be applied. A media query takes max or min height or width. They may look like this:

@media (max-height: 400px) {
h4 {
font-size: 12px;
}
}

This media query basically says, that for viewports with a maximum height of 400px, make the h4 header have a font-size of 12px.

Pixel density, which can impact how high resolution your page is displayed, can vary between devices. They have different pixel density to Non-Retina displays which often have lower resolution. Images not made with high-DPI/high-resolution in mind will end up looking pixelated. Typography Responsive

We have been using px and em to size our text and elements for that matter, but we can also use 'viewport units' which function similarly to percentages in that they are relative units, but of course are different. "Viewport units are relative to the viewport dimensions (width or height) of a device, whereas percentages are relative to the size of the parent container element." - Free Code Camp, 2020 The four viewport dimensions include:

  • vw (viewport width): 15vw would be 15% of the viewport's width.
  • vh (viewport height): 40vh would be 40% of the viewport's height.
  • vmin (viewport minimum): 9vmin would be 9% of the viewport's smaller dimension (height or width).
  • vmax (viewport maximum): 100vmax would be 100% of the viewport's bigger dimension (height or width).

CSS Flexbox

CSS3 introduced Flexible Boxes, or flexbox, to create page layouts for a dynamic UI. It is a layout mode that arranges elements in a predictable way for different screen sizes and browsers. While somewhat new, all popular modern browsers support flexbox. This section covers how to use flexbox and the different layout options it offers.

Here are a few new flexbox options:

  • 'flexdirection' - Takes 'row', 'column', 'row-reverse' and 'column-reverse' as values.
  • 'justify-content' - Aligns items along the main axis.(REWORD THIS LESSON HERE)
  • 'align-items' - Aligns items on the cross axis.(REWORD LESSON HERE)
  • 'flex-wrap' - Fit items into the same row and move extra items to a new row. This takes 'nowrap' (default), 'wrap' and 'reverse wrap'.
  • 'flex-shrink' - Causes an item to shrink in the container if the container is too small to fit them all. The numerical value it takes denotes by how many times the item can shrink.
  • 'flex-grow' - The inverse of 'flex-shrink'
  • 'flex-basis' - Specifies initial size of the item before flex-shrink/grow adjustments. It takes all size properties, auto lets it size the item based on its content size.
  • 'flex' - Property. Uses clockwise notation to take values for 'flex-shrink', 'flex-grow' and lastly, 'flex-basis'.
  • 'order' - Property. Denotes the order from left-right/top-down the flex items appear.
  • 'align-self' - Property. Allows you to adjust each item's alignment individually, instead of setting them all at once. Takes the same values as 'align-items' and will also override it. This is a good work around for other common adjustment techniques using the CSS properties float, clear, and vertical-align which do not work on flex items.

CSS Grid

The CSS grid allows you to design web pages easily and more simply. Basically, it turns an HTML element into a grid with rows and columns which you can then use to place child elements into exactly where you want them to be in this grid. The grid is made up of lines with the top line valued as line '1' and and the edge of its row or column begins line '2'. This image may help explain the grid better:


Here are some useful grid tools:

  • 'display:grid' - Establishes your grid within the element you put this in. You can also create a grid within a grid by using this on an item of your main grid.
  • 'grid-template-columns'/'grid-template-rows' - Adds columns/rows to your grid.
  • 'fr' - This value sets the column or row to be a fraction of the available space
  • 'auto' - This value sets the column or row to the width/height of its content automatically
  • '%' - Adjusts the column or row to the percent width of its container
  • 'grid-column-gap' - Places a gap between the columns. According to VS code, this is now obsolete, however it still works for me on Firefox as of 3/5/2020.
  • 'grid-row-gap' - Places a gap between rows
  • 'grid-gap' - Short-form for previous 2. Takes up to 2 values where the first value is taken for rows, and the second is for columns.
  • 'grid-column'/'grid-row' - Allows us to specify how many columns/rows an item will consume in the grid. If written as 'grid-columm: 1/2', the item will fill the first column as it starts at line one and ends at line two.
  • 'justify-self' - Align the content's position within its cell (box) horizontally. By default this is set to 'stretch' which makes the item fill the cell. Other values include 'start' (aligns left), 'center' and 'end' (aligns right). You can use this in your parent container to set the alignment for all items, or individually.
  • 'align-self'- Aligns the item within its cell vertically. Takes the same values as 'justify-self'.

Grid Area

  • 'grid-template-areas' - Allows you to group cells of your grid together into an area and give the area a custom name. Every word in the code represents a cell and every pair of quotation marks represent a row. In addition to custom labels, you can use a period (.) to designate an empty cell in the grid.
  • 'grid-area' - Once you've made your area template, use this to assign the name to your item. Or you can use this with line designation which will look like: 'grid-area: 1/2/3/4' - this translates to 'The item will now sit in the space starting at horizontal line 1 to horizontal line 3, and starting at vertical line 2 and ending at vertical line 4'.

Repeat

Additionally, to create this many columns and rows without having to provide size values for all of them using the 'grid-template-columns' and 'grid-template-rows', you can use 'grid-template-columns'/'grid-template-rows' with the attribute 'repeat' followed by in parentheses your preferred number of columns/rows followed by another value still in these parentheses to indicate the size. This will look like: 'grid-template-columns: repeat(20, 30px);'. This means the grid will have 20 columns all at 30px in width. You can also put multiple values as shown below: You can also use 'grid-template-columns'/'grid-template-rows' with 'minmax' which will limit the size of items when the grid container changes size. It would look something like this 'grid-template-columns: repeat(3, minmax(90px, 1fr))'. Further interesting things include:

  • 'auto-fill' - This allows you to automatically insert as many rows or columns of your desired size as possible depending on the size of the container. You can create flexible layouts when combining auto-fill with minmax.
  • 'auto-fit' - Similar to 'auto-fill' except that 'auto-fill' will keep adding rows/columns to fill the container even if items don't fill it up. 'auto-fit' makes them stretch to fit the remainder of the container.
