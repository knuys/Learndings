Basic HTML 5

New Elements and Attributes

While I am revisiting old concepts which may be viewed on my Codecademy: Make A Website page, I am also introduced to a few new HTML elements, attributes and types. These include:

  • Anchors - element; finally I can create internal links to different sections of my webpage which can be observed on this page. To do this, I created an anchor using the 'a' element, which the 'href' then takes a # with an 'id' attribute which is assigned to an element I wish to link to. You can also create 'dead links' using just the # symbol, this is useful when modifying the link in Javascript, or as a placeholder if the link is yet unknown. You can also use dead links to turn an image into a clickable link, by nesting the image in the dead anchor.
  • target="_blank" - this attribute/value coupling can be used to open links in a fresh tab.
  • input - a self-closing element that allows me to get input from a user. I can select a type of input accepted by it aswell such as text
  • Placeholder - attribute; placeholder text is what sits in the input box before the user has put anything in.
  • Form - element; allows you to build webform and submit input data to a server. To do so, use the 'action' attribute.
  • Button - self-explanatory element; this also takes a 'type' and text.
  • Required - attribute; can be given to 'input' so inputs are...well..required to progress in a form for example.
  • Label - element; nesting elements within it, will associate them with the label element and can lead to cool things like clickable associated text for radio buttons. In this case, it is always good practice to give your labels a 'for' attribute whose label matches with the input 'id' attribute. Each label in a set should share the same 'name' attribute value in the input element to create a radio group.
  • Radio buttons - 'type' value of 'input'. Basically multiple-choice buttons. Below I included ones with and without 'label'.
  • Checkboxes - a type of input. It is best practice to match your label 'for' and input 'id' values.

We also touch on creating ordered and unordered lists, which I love using. The example on Free Code Camp was to create lists on what cats love and hate. I decided to use my favourite image of a cat displaying the 'hatred' emotion state to utilise the 'dead linking' I mentioned earlier.

Basic CSS

Again, a lot of this is a refresh of old concepts such as setting fonts, changing colours, padding and margins, however there are a few new tricks as well. Here are some new attributes and selectors:

  • font-family - to specify/call the font you want. You can import font by linking to them above your style block. Be sure to remember to declare 'rel' and 'type'. Family names should be in "" if they have spaces
  • Class - attribute; we used it before, but we can pile on classes just by using a space in the declaration!
  • Attribute Selectors - using this to style elements by using [attribute="value"] as a selector. I did this with my radio buttons to give them a margin.
  • border-radius - to create rounded border edges. Giving it a border radius of 50% will create a round image. I used this for banana cat.
  • Background-color - used this for my nav section!
  • 'Value' - attribute; because checkboxes and radio report their values from the 'value' attribute, we should define what it is when the corresponding boxes/buttons are selected. The default would be 'on' (used when no value attribute has been assigned), however this won't be useful if we have a long list of buttons as this would appear as 'input-name: on', rather than the id or text the user had selected.
  • Checked - attribute; you can use this to 'check' (fill in) by default a radio or checkbox button. I've used this on a couple buttons below - I don't generally like them for surveys, but I'm implementing it below anyway. It has a good use case for things like pre-input/saved user data.
  • id - attribute; it's considered best practice to give elements unique id's. They are useful for selecting and modifying specific elements using Javascript. I did this for my short meme survey. We reference id using # rather than a dot operator like classes

Formatting and Overrides

There was also some general information on formatting and style overrides:

  • CSS class declaration in HTML - so I don't think I did this before in my previous workshop, however this is pretty neat. Basically I can create a CSS class and use it in a style block to modify HTML elements. I did this to create an orange heading for this segment.
  • Clockwise-notation - I have been doing this almost religiously so I don't have to specify -top, -right, -bottom, -left for margins, borders and well, anything really, by inputting all of my values into a single, say, 'padding' with the values corresponding to the directions going clockwise from top to left.
  • Declaration override - If you have competing styles in your class, the second declaration will always take precedence over the first. 'id' attribute, then inline styling will override everything competing with it.
  • !important- Makes sure that the specific style is applied and not overridden by other styles for example. I had to use this for my radio button's margins because Bootstrap was interfering.
  • Short Hex Codes - It's a thing. Instead of writing #FF0000 for red, you could just write #F00...or red for that matter.

Variables and Root

Variables

Variables in CSS are denoted using '--' before the variable name. This would allow you to change multiple style properties at once. You can reference your variable using 'var(--variable-name)' as a value to a property. You may also assign a Fallback value.This will be displayed if the desired value or variable could not be found. Good for debugging or compatibility. This can look like:

  • background:var(--variable-name, fallback);
  • or simply background:red; if placed the line above the variable.

Root

When you create a variable, it is available for you to use inside the selector in which you create it. It also is available in any of that selector's descendants. This happens because CSS variables are inherited, just like ordinary properties. To make use of inheritance, CSS variables are often defined in the :root element. :root is a pseudo-class selector that matches the root element of the document, usually the html element. By creating your variables in :root, they will be available globally and can be accessed from any other selector in the style sheet. Define a variable named --penguin-belly in the :root selector and give it the value of pink. You can then see that the variable is inherited and that all the child elements which use it get pink backgrounds. When you create your variables in :root they will set the value of that variable for the whole page. You can then over-write these variables by setting them again within a specific element.

Media Queries

CSS Variables can simplify the way you use media queries. For instance, when your screen is smaller or larger than your media query break point, you can change the value of a variable, and it will apply its style wherever it is used. In the :root selector of the media query, change it so --penguin-size is redefined and given a value of 200px. Also, redefine --penguin-skin and give it a value of black. Then resize the preview to see this change in action.

We are also learning about new tags which apply CSS when used in HTML. These include:

  • Strong - using this in your HTML will apply the CSS font-style: bold; to whatever it is wrapped around
  • U - underlines what it is wrapped around, applying the CSS text-decoration: underline;
  • Em - font-style: italics
  • S - text-decoration: line-through
  • hr - self-closing; adds a horizontal line to separate elements in HTML4, but in HTML5 it's more like a break

A few other things like adding a to rgb --> as in rgba() will allow us to adjust the colour opacity.

Box Shadows

These add nifty little shadows to an element. Multiple box shadows can be created by using commas to separate properties of each box shadow element.This property takes the following values in this order:

  • offset-x - how far to push the shadow horizontally from the element
  • offset-y - how far to push the shadow vertically from the element
  • blur-radius - optional; how blurry the shadow is with 0 = sharp and higher numbers or percentages increase the amount of blur
  • spread-radius - optional; size of the shadow. The default is 0 in which the spread-radius is equal to the blur size
  • color - shadow color

Even More Properties

  • Opacity - This sets the opacity of items. Takes a value of 0-1
  • text-transform - Turns case into whatever the value is such as lowercase, uppercase, capitalise, initial (default value), inherit (parent value) and none (original text).
  • font-weight - Sets how thicc characters are
  • line-height - Sets the line height between lines

  • hover - this selects for what happens to the element if the mouse is hovering over it. You can observe this in the Animal Crossing card where the link hover is red.

Position

We can use this to override the CSS box-model and place items where we would like them. We will be looking at the following 4 items:

  • Relative - this allows placement of elements in a spot relative to the normal flow in the box model. This uses the CSS offset properties of left or right, and top or bottom to say how far away from the normal position the element will now be.
  • Absolute - this locks an element into a position relative to the parent container. This takes the element out of the normal flow of the document making surrounding items ignore it. Basically, it's out here doing its own thing and everyone else is none the wiser. The CSS offset properties are used to adjust the position.

    • Be careful! Absolute positioning can be problematic. If you don't add a position rule to your parent, your browser will look 'up the chain' to prescribe your 'absolute'. It could result in defaulting to the body tag to receive that info.

  • Fixed - a type of absolute position. This locks an element into place relative to the browser window and takes CSS offset properties. Fixed allows you to have something stay in the same position even when you scroll. However, be careful with styling - as it is removed from the normal flow of the document, it can disrupt the aesthetics of the page resulting in the need for layout adjustments/accommodations. It doesn't occupy a 'space' per se and just hangs around on a spot on the viewport - like a fly on your screen.
  • Sticky - a type of relative position similar to fixed. However, it does occupy a space in a container, and acts similar to fixed once it is scrolled past. You can observe this below in my triptych of my favourite animal crossing friends.

As mentioned earlier, we may, for aesthetic or functional reasons, want to make adjustments to our layout to prevent or reduce overlap. z-index allows us to specify the order of how our elements are stacked. This property takes an integer whereby elements with higher values will appear at the top than those with lower values.
